# Contributing to everything-imu

## Workflow

1. Branch from `main`: `git checkout -b feat/short-name`
2. Conventional commits (see below)
3. PR to `main`, ensure CI green
4. At least one review before merge (self-review acceptable for solo phase)

## Commit message style

Conventional commits with scope:

```
<type>(<scope>): <subject>

[optional body]
[optional footer(s)]
```

| Type | Use |
|------|-----|
| `feat` | New feature |
| `fix` | Bug fix |
| `refactor` | Code restructure, no behavior change |
| `chore` | Build, deps, tooling |
| `docs` | Documentation only |
| `test` | Test additions / fixes |
| `perf` | Performance improvement |
| `ci` | CI/CD config |

Scopes: `tracker`, `fusion`, `joycon`, `dualsense`, `dualshock3`, `psmove`, `wii`, `3ds`, `vita`, `steam`, `hopx`, `tesla`, `companions`, `core`, `ui`, `tauri`, `persistence`, `protocol`.

Examples:

```
feat(joycon): add JC2 magnetometer 9D fusion path
fix(tracker): BUNDLE auto-fallback when server lacks FEATURE_FLAGS
refactor(core): extract device registry into separate module
chore(deps): bump btleplug to 0.11.5
```

## Code style

### Rust

- `cargo fmt` (rustfmt settings in `rustfmt.toml`)
- `cargo clippy --all-targets -- -D warnings` must pass
- Prefer `?` over `.unwrap()` outside tests
- `tracing` for logs, never `println!` in library code
- Module organization: one struct/trait per file when > 200 LOC

### TypeScript / React

- Biome for lint + format
- React components: function components, no classes
- Hooks: `use*` naming
- Zustand stores in `apps/ui/src/stores/`
- IPC types: never write by hand, always generated by `tauri-specta`

### Tests

- Rust: `#[test]` in same file for unit, `tests/` dir for integration
- Wire-compat tests in `crates/slime-tracker/tests/` are **non-negotiable** â€” never skip
- Property tests with `proptest` for fusion math

## PR checklist (suggested)

- [ ] Conventional commit style
- [ ] `cargo fmt` + `cargo clippy` clean
- [ ] Tests added/updated
- [ ] Wire-compat preserved (if touching `slime-tracker`)
- [ ] Documentation updated if behavior changed
- [ ] CHANGELOG entry if user-visible

## Reporting bugs / requesting features

Use GitHub Issues with templates:
- Bug report: include OS, app version, device, log excerpt
- Feature request: include use case + acceptance criteria

For SlimeVR protocol questions, cross-reference `docs/PROTOCOL.md` first.

## Development setup

### Prerequisites

- Rust stable (matches `rust-toolchain.toml`)
- Node.js 22+ + pnpm 10
- Tauri 2 prerequisites per OS:
  - Windows: WebView2 (preinstalled Win11)
  - Linux: `webkit2gtk-4.1`, `libgtk-3-dev`, `libsoup-3.0-dev`, `librsvg2-dev`, `libudev-dev`

### Build

```bash
# Workspace check
cargo check --workspace

# Run Tauri dev server
pnpm install
pnpm tauri dev

# Production bundle
pnpm tauri build
```

### Tests

```bash
# Workspace tests
cargo test --workspace

# UI tests
pnpm typecheck
pnpm lint
```
